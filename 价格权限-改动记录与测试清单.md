# 价格权限整改 — 改动记录与测试清单

> 目标：员工(role_2)不能看到采购价/costPrice/unitPrice，管理员(role_1)不受影响。
> 改动日期：2026-06-09

---

## 一、改动策略

- **后端**：Service 层判断角色，非管理员返回数据时价格字段置 `null`；敏感接口 Controller 层加 `@SaCheckRole("role_1")` 直接拦截
- **前端**：复用已有的 `userStore.isAdmin` + 路由 `requireAdmin` 机制，条件渲染价格相关 UI
- **售价(salePrice)**：不做限制，维持现状

---

## 二、后端改动明细

### 2.1 ProductService.java
**路径**: `inventory-server/.../product/service/ProductService.java`

- 新增 `SysUserMapper` 依赖注入
- 新增 `private boolean isAdmin()` 方法
- `page()`: 非管理员忽略 `minPrice`/`maxPrice` 筛选参数，返回结果 `purchasePrice` 置 null
- `list()` / `listAll()` / `listFiltered()`: 非管理员 `purchasePrice` 置 null
- `getById()`: 非管理员 `purchasePrice` 置 null

### 2.2 ProductController.java
**路径**: `inventory-server/.../product/controller/ProductController.java`

- `export()`: 加 `@SaCheckRole("role_1")` — 仅管理员可导出商品
- `exportForImport()`: 加 `@SaCheckRole("role_1")` — 仅管理员可导出导入格式

### 2.3 PurchaseOrderController.java
**路径**: `inventory-server/.../purchase/controller/PurchaseOrderController.java`

- `page()`: 加 `@SaCheckRole("role_1")` — 仅管理员可查看采购订单列表
- `getById()`: 加 `@SaCheckRole("role_1")` — 仅管理员可查看采购订单详情
- `create()`: 加 `@SaCheckRole("role_1")` — 仅管理员可创建采购单
- `submit()`: 加 `@SaCheckRole("role_1")` — 仅管理员可提交采购单
- `updateDraft()`: 加 `@SaCheckRole("role_1")` — 仅管理员可修改草稿

### 2.4 InventoryService.java
**路径**: `inventory-server/.../inventory/service/InventoryService.java`

- 新增 `SysUserMapper` 依赖注入
- 新增 `private boolean isAdmin()` 方法
- `page()` / `listAll()` / `listByWarehouse()`: 非管理员 `costPrice` 置 null

### 2.5 InventoryController.java
**路径**: `inventory-server/.../inventory/controller/InventoryController.java`

- 新增 import `SaCheckRole`
- `export()`: 加 `@SaCheckRole("role_1")` — 仅管理员可导出库存

### 2.6 TransferService.java
**路径**: `inventory-server/.../transfer/service/TransferService.java`

- 新增 `private boolean isAdmin()` 方法（调拨明细实体本身无 unitPrice 字段，预留方法供后续使用）
