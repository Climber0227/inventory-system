## 七、入库按日期批次分开（不再累加）

### 后端
| 文件 | 方法 | 改动 |
|------|------|------|
| `PurchaseOrderService` | `approve()` | 每次入库 ALWAYS INSERT 新行，自动生成批次号 `IN{date}`，不再 UPDATE 旧行 |
| `PurchaseOrderService` | `cancel()` | 删除旧回滚逻辑（已在第六条去掉） |
| `SalesOrderService` | `cancel()` | 同上 |
| `TransferService` | `approve()` 源仓库 | 优先按 `batchNo` 精确扣减，否则后进先出 |
| `TransferService` | `approve()` 目标仓库 | 每次调入 ALWAYS INSERT 新行，批次号 `TR{date}` |
| `TransferService` | `cancel()` | 删除旧回滚逻辑 |
| `StockTakeService` | `create()` | 全盘按批次生成明细（每批次一行） |
| `StockTakeService` | `addItem()` | 支持 `batchNo`，按批次取账面数 |
| `StockTakeService` | `adjust()` | 使用审批时保存的 `diffQty`，按 `batchNo` 精确匹配批次独立调整 |

### 成本价改为入库采购价
| 文件 | 改动 |
|------|------|
| `PurchaseOrderService.approve()` | 新批次 `costPrice` = 入库采购单价（`item.getUnitPrice()`），不再加权平均 |
| `TransferService.approve()` | 新批次 `costPrice` = 源批次成本价（按 `batchNo` 精确匹配） |
