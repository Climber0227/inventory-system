# 交付前 Bug 修复总结

日期: 2026-05-22

## 1. 条码加载失败（员工角色）

**问题**：`BarcodeController.java` 类级别标注了 `@SaCheckRole("role_1")`，员工访问商品列表条码时返回 403。

**修复**：移除类级别注解，条码生成是工具功能，不涉及敏感数据，所有角色均可使用。

## 2. 取消已入库/已出库单据权限缺失

**问题**：员工可以取消已影响库存的单据（CONFIRMED 状态），涉及库存回滚，存在数据安全风险。

**修复**：
- 前端：`PurchaseList.vue` / `SalesList.vue` 的「取消入库/取消出库」按钮仅管理员可见
- 后端：PurchaseOrderService / SalesOrderService / TransferService 的 CONFIRMED 状态取消逻辑增加管理员角色校验

## 3. 分类管理权限缺失

**问题**：`CategoryController` 的 4 个写操作接口（新增/修改/删除/恢复）无 `@SaCheckRole`，任何登录用户均可操作。

**修复**：
- 后端：4 个接口加 `@SaCheckRole("role_1")`
- 前端：`CategoryList.vue` 的新增/编辑/作废按钮加 `v-if="userStore.isAdmin"`

## 4. 系统配置更新接口权限缺失

**问题**：`SysConfigController` 的 PUT 更新接口无 `@SaCheckRole`，任何登录用户可修改系统全局配置。

**修复**：加 `@SaCheckRole("role_1")`

## 5. 采购/销售导出接口权限缺失

**问题**：采购订单和销售订单的导出接口无角色限制，暴露全部业务明细数据（含金额）。

**修复**：两个导出接口加 `@SaCheckRole("role_1")`

## 6. CORS 配置过于宽松

**问题**：`WebMvcConfig.java` 中 `addAllowedOriginPattern("*")` + `setAllowCredentials(true)`，任意第三方网站在用户已登录时可发送跨域请求。

**注意**：生产环境建议改为具体域名白名单，或在 Nginx 层面做同源限制。
