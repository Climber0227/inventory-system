# 交付前 Bug 修复总结 + 生产部署指南

日期: 2026-05-22

## 一、本次修复的 Bug

### 1.1 条码加载失败（员工角色）

**问题**：`BarcodeController.java` 类级别标注了 `@SaCheckRole("role_1")`，员工访问商品列表条码时返回 403。

**修复**：移除类级别注解，条码生成是工具功能，不涉及敏感数据，所有角色均可使用。

### 1.2 取消已入库/已出库单据权限缺失

**问题**：员工可以取消已影响库存的单据（CONFIRMED 状态），涉及库存回滚，存在数据安全风险。

**修复**：
- 前端：`PurchaseList.vue` / `SalesList.vue` 的「取消入库/取消出库」按钮仅管理员可见
- 后端：PurchaseOrderService / SalesOrderService / TransferService 的 CONFIRMED 状态取消逻辑增加管理员角色校验

### 1.3 分类管理权限缺失

**问题**：`CategoryController` 的 4 个写操作接口（新增/修改/删除/恢复）无 `@SaCheckRole`，任何登录用户均可操作。

**修复**：
- 后端：4 个接口加 `@SaCheckRole("role_1")`
- 前端：`CategoryList.vue` 的新增/编辑/作废按钮加 `v-if="userStore.isAdmin"`

### 1.4 系统配置更新接口权限缺失

**问题**：`SysConfigController` 的 PUT 更新接口无 `@SaCheckRole`，任何登录用户可修改系统全局配置。

**修复**：加 `@SaCheckRole("role_1")`

### 1.5 采购/销售导出接口权限缺失

**问题**：采购订单和销售订单的导出接口无角色限制，暴露全部业务明细数据（含金额）。

**修复**：两个导出接口加 `@SaCheckRole("role_1")`

### 1.6 CORS 配置过于宽松

**问题**：`WebMvcConfig.java` 中 `addAllowedOriginPattern("*")` + `setAllowCredentials(true)`，任意第三方网站在用户已登录时可发送跨域请求。

> **注意**：生产环境建议改为具体域名白名单，或在 Nginx 层面做同源限制。

## 三、生产部署指南

### 3.1 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17+ | Spring Boot 3.2 要求 |
| MySQL | 8.0+ | 数据库 |
| Nginx | 1.20+ | 反向代理 + 静态文件服务 |
| 服务器 | 2 核 4G 起 | 4 核 8G 推荐 |

### 3.2 部署步骤

#### 第一步：服务器准备

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker（如使用 Docker 部署）
curl -fsSL https://get.docker.com | sh

# 安装 Java（如手动部署）
apt install openjdk-17-jdk -y

# 安装 Nginx（如手动部署）
apt install nginx -y

# 安装 MySQL（如手动部署）
apt install mysql-server -y
```

#### 第二步：获取代码

```bash
git clone https://github.com/Climber0227/inventory-system.git /data/inventory
cd /data/inventory
```

#### 第三步：选择部署方式

**方式一：Docker 部署（推荐）**

```bash
# 1. 修改 docker-compose.yml 中的密码
#    - MYSQL_ROOT_PASSWORD: 修改默认密码 root123
#    - SPRING_DATASOURCE_PASSWORD: 与上方一致
# 2. 创建数据持久化目录
mkdir -p data/mysql data/uploads
# 3. 启动
docker-compose up -d
# 4. 查看运行状态
docker-compose ps
docker-compose logs -f
```

**方式二：手动部署**

```bash
# 1. 初始化数据库
mysql -u root -p < sql/01-schema.sql

# 2. 创建 application-prod.yml 配置文件
cp inventory-server/src/main/resources/application.yml inventory-server/src/main/resources/application-prod.yml
```

编辑 `application-prod.yml`，修改以下内容：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/inventory?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&nullCatalogMeansCurrent=true
    username: inventory       # 建议创建专用账号，不使用 root
    password: 你的数据库密码

sa-token:
  jwt-secret-key: 替换为随机字符串，至少32位
  timeout: 86400              # Token 有效期（秒）
  active-timeout: 1800        # 活跃超时（秒）

app:
  upload-base-path: /data/inventory/uploads   # 上传文件路径

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.nologging.NoLoggingImpl  # 关闭 SQL 日志

logging:
  level:
    com.inventory: warn
```

```bash
# 3. 创建上传目录
mkdir -p /data/inventory/uploads/images

# 4. 编译后端
cd inventory-server
mvn clean package -DskipTests
java -jar target/inventory-server-*.jar --spring.profiles.active=prod &

# 5. 编译前端
cd ../inventory-admin
npm install
npm run build        # 产物在 dist/

# 6. 部署前端
cp -r dist/* /data/inventory/admin/
```

#### 第四步：配置 Nginx

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    client_max_body_size 10m;

    location / {
        root /data/inventory/admin;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
}
```

#### 第五步：配置 HTTPS（推荐）

```bash
# 使用 Certbot 申请 Let's Encrypt 证书
apt install certbot python3-certbot-nginx -y
certbot --nginx -d 你的域名
```

#### 第六步：配置数据库备份

创建 `/data/inventory/backup.sh`：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d)
mysqldump -u inventory -p密码 inventory > /data/inventory/backup/inventory_$DATE.sql
find /data/inventory/backup -mtime +30 -name "*.sql" -delete
```

添加定时任务：

```bash
chmod +x /data/inventory/backup.sh
crontab -e
# 添加：0 3 * * * bash /data/inventory/backup.sh
```

### 3.5 常见问题

**Q：部署后访问显示 502 Bad Gateway**
A：检查后端是否启动成功，`java -jar` 启动日志是否有报错。确认 Nginx `proxy_pass` 指向的地址端口正确。

**Q：前端页面白屏/路由跳转404**
A：确认 Nginx 配置了 `try_files $uri $uri/ /index.html;`，这是 Vue 路由必需的。

**Q：图片上传后无法访问**
A：检查 `uploads/` 目录是否存在且有写入权限。确认 Nginx 的 `/uploads/` 反向代理配置正确。

**Q：小程序打开空白**
A：确认 `BASE_URL` 填的是公网可访问的地址。微信开发者工具中勾选「不校验合法域名」。

**Q：数据库连接失败**
A：确认 MySQL 服务已启动，用户名密码正确，数据库 `inventory` 已创建。Docker 部署时确认容器网络正常。
