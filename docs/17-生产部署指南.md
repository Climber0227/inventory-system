# 生产部署指南

日期: 2026-05-22

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17+ | Spring Boot 3.2 要求 |
| MySQL | 8.0+ | 数据库 |
| Nginx | 1.20+ | 反向代理 + 静态文件服务 |
| 服务器 | 2 核 4G 起 | 4 核 8G 推荐 |

## 2. 部署步骤

### 2.1 服务器准备

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker（如使用 Docker 部署）
curl -fsSL https://get.docker.com | sh

# 安装 Java（如手动部署）
apt install openjdk-17-jdk -y

# 安装 Nginx（如手动部署）
apt install nginx -y

# 安装 MySQL（如手动部署）
apt install mysql-server -y
```

### 2.2 获取代码

```bash
git clone https://github.com/Climber0227/inventory-system.git /data/inventory
cd /data/inventory
```

### 2.3 部署方式一：Docker（推荐）

```bash
# 1. 修改 docker-compose.yml 中的密码
#    - MYSQL_ROOT_PASSWORD: 修改默认密码 root123
#    - SPRING_DATASOURCE_PASSWORD: 与上方一致

# 2. 创建数据持久化目录
mkdir -p data/mysql data/uploads

# 3. 启动
docker-compose up -d

# 4. 查看运行状态
docker-compose ps
docker-compose logs -f
```

### 2.4 部署方式二：手动部署

**1. 初始化数据库**

```bash
mysql -u root -p < sql/01-schema.sql
```

**2. 创建生产配置文件**

```bash
cp inventory-server/src/main/resources/application.yml inventory-server/src/main/resources/application-prod.yml
```

编辑 `application-prod.yml`，修改以下关键配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/inventory?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&nullCatalogMeansCurrent=true
    username: inventory       # 建议创建专用账号，不使用 root
    password: 你的数据库密码

sa-token:
  jwt-secret-key: 替换为随机字符串，至少32位
  timeout: 86400              # Token 有效期（秒），24 小时
  active-timeout: 1800        # 活跃超时（秒），30 分钟

app:
  upload-base-path: /data/inventory/uploads   # 上传文件存储路径

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.nologging.NoLoggingImpl  # 关闭 SQL 日志

logging:
  level:
    com.inventory: warn        # 生产环境降低日志级别
```

**3. 创建上传目录**

```bash
mkdir -p /data/inventory/uploads/images
```

**4. 编译并启动后端**

```bash
cd inventory-server
mvn clean package -DskipTests
java -jar target/inventory-server-*.jar --spring.profiles.active=prod &
```

**5. 编译并部署前端**

```bash
cd ../inventory-admin
npm install
npm run build        # 产物在 dist/
cp -r dist/* /data/inventory/admin/
```

### 2.5 配置 Nginx

创建 `/etc/nginx/sites-available/inventory`：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    client_max_body_size 10m;

    location / {
        root /data/inventory/admin;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
}
```

```bash
ln -s /etc/nginx/sites-available/inventory /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

### 2.6 配置 HTTPS（推荐）

```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d 你的域名
```

### 2.7 配置数据库备份

创建 `/data/inventory/backup.sh`：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d)
BACKUP_DIR=/data/inventory/backup
mkdir -p $BACKUP_DIR
mysqldump -u inventory -p密码 inventory > $BACKUP_DIR/inventory_$DATE.sql
find $BACKUP_DIR -mtime +30 -name "*.sql" -delete
```

```bash
chmod +x /data/inventory/backup.sh
crontab -e
# 添加：0 3 * * * bash /data/inventory/backup.sh
```

## 5. 常见问题

**Q：部署后访问显示 502 Bad Gateway**
A：检查后端是否启动成功，`java -jar` 启动日志是否有报错。确认 Nginx `proxy_pass` 指向的地址端口正确。

**Q：前端页面白屏/路由跳转404**
A：确认 Nginx 配置了 `try_files $uri $uri/ /index.html;`，这是 Vue 路由必需的。

**Q：图片上传后无法访问**
A：检查 `uploads/` 目录是否存在且有写入权限。确认 Nginx 的 `/uploads/` 反向代理配置正确。

**Q：小程序打开空白**
A：确认 `BASE_URL` 填的是公网可访问的地址。微信开发者工具中勾选「不校验合法域名」。

**Q：数据库连接失败**
A：确认 MySQL 服务已启动，用户名密码正确，数据库 `inventory` 已创建。Docker 部署时确认容器网络正常。
