# 生产部署指南（零基础版）

> 本教程面向没有服务器经验的用户。跟着步骤一步步走，遇到问题先看章节末尾的「常见错误」。

---

## 准备工作

### 你需要什么

1. **一台云服务器**（阿里云、腾讯云、华为云都行）
   - 配置：2核CPU、4GB内存、40GB硬盘（最低）
   - 系统：Ubuntu 24.04 LTS
   - 买的时候选"按量付费"也行，部署完再转包年

2. **一个域名**（可选但推荐）
   - 小程序要求已备案域名 + HTTPS，必须有
   - Web 管理后台可以只用 IP

3. **一台能上网的电脑**
   - Windows / Mac 都行

### 第一步：购买服务器

以阿里云为例：
1. 登录 aliyun.com
2. 搜索"ECS" → 点「创建实例」
3. 地区选离你最近的
4. 镜像选 **Ubuntu 24.04**
5. 规格选 **2 vCPU 4 GiB**
6. 系统盘 40GB
7. 勾选「分配公网 IPv4 地址」
8. 设置登录密码（**记住这个密码！**）
9. 点击「创建」

创建完成后，你会看到一个 **公网 IP**（类似 `47.96.xxx.xxx`），记下来。

## 连接服务器

### Windows 用户

打开 PowerShell（右键开始菜单 → Windows PowerShell），输入：

```bash
ssh root@你的服务器IP
```

第一次会提示"是否信任"，输入 `yes` 回车。然后输入创建服务器时设的密码（输入时不显示，正常）。

### Mac 用户

打开「终端」（在启动台搜索 Terminal），输入同样的命令：

```bash
ssh root@你的服务器IP
```

连上后你会看到类似 `root@xxx:~#` 的提示符，这就对了。

## 开始部署

以下所有命令都在服务器上执行，**一条一条复制粘贴**。

### 1. 更新系统

```bash
apt update && apt upgrade -y
```

这行命令的作用是更新系统软件包，让服务器处于最新状态。大概需要 2-3 分钟。

### 2. 安装 Docker

```bash
curl -fsSL https://get.docker.com | sh
```

这条命令自动下载和安装 Docker。装完后把当前用户加入 docker 组，以后不用每次都加 `sudo`：

```bash
usermod -aG docker root
```

退出再重新连接让配置生效：

```bash
exit
```

（等 3 秒，再用 `ssh root@你的IP` 重新连）

验证安装：

```bash
docker --version
```

显示 `Docker version xx.xx.x` 就说明成功了。

### 3. 下载项目代码

```bash
mkdir -p /data
cd /data
git clone https://github.com/Climber0227/inventory-system.git inventory
cd inventory
```

如果报 `git: command not found`，先装 Git：

```bash
apt install git -y
```

然后重新执行 `git clone`。

### 4. 修改密码（重要！）

用编辑器打开配置文件：

```bash
nano docker-compose.yml
```

方向键移动到以下两行，把 `root123` 改成**你自己设的密码**（8 位以上，包含字母和数字）：

```yaml
MYSQL_ROOT_PASSWORD: root123    ← 改这里
...
SPRING_DATASOURCE_PASSWORD: root123  ← 也改这里，保持一致
```

改完之后：
1. 按 `Ctrl + X`
2. 按 `Y`（确认保存）
3. 按 `Enter`（确认文件名）

### 5. 创建数据目录

```bash
mkdir -p data/mysql data/uploads
```

### 6. 启动系统

```bash
docker compose up -d
```

这行命令会：
- 下载 MySQL、Java、Nginx 等运行环境
- 编译后端代码
- 编译前端代码
- 启动所有服务

**第一次启动大约需要 5-15 分钟**，取决于网速。看到类似 `Started inventory-server` 就说明启动好了。

查看启动进度：

```bash
docker compose logs -f
```

看到滚动日志后，按 `Ctrl + C` 退出。

### 7. 验证部署

在浏览器打开：`http://你的服务器IP`

看到登录页面就说明部署成功了。默认账号密码：

| 账号 | 密码 | 角色 |
|------|------|------|
| admin | 123456 | 管理员（全部权限） |
| warehouse | 123456 | 仓管员 |
| sales | 123456 | 销售员 |

## 配置域名和 HTTPS

> 如果你只需要 Web 管理后台（不需要小程序），可以跳过这步，直接用 IP 访问。

### 1. 域名解析

在域名服务商（阿里云/腾讯云等）的控制台，添加一条 A 记录：

| 主机记录 | 记录类型 | 记录值 |
|---------|---------|--------|
| @（或 www） | A | 你的服务器 IP |

### 2. 申请 HTTPS 证书

域名解析生效后（通常 5-10 分钟），在服务器上执行：

```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d 你的域名
```

按提示输入邮箱、同意协议即可。证书会自动续期。

## 小程序部署

### 1. 确保有 HTTPS 域名

微信小程序生产环境要求：
- 已备案域名
- HTTPS 协议
- 在小程序后台配置服务器域名白名单

### 2. 修改 API 地址

在电脑上打开项目目录，找到 `inventory-miniapp/src/api/request.js`，第 2 行：

```js
const BASE_URL = 'http://192.168.10.162:8080/api/v1'
```

改成：

```js
const BASE_URL = 'https://你的域名/api/v1'
```

### 3. 编译上传

```bash
cd inventory-miniapp
npm install
npm run build:mp-weixin
```

用微信开发者工具打开 `dist/build/mp-weixin/`，上传审核。

## 日常维护

### 查看运行状态

```bash
cd /data/inventory
docker compose ps
```

### 查看日志

```bash
docker compose logs -f
```

### 更新代码（有新版本时）

```bash
cd /data/inventory
git pull
docker compose down
docker compose up -d --build
```

### 配置数据库自动备份

创建备份脚本：

```bash
cat > /data/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d)
mkdir -p /data/backup
docker exec inventory-mysql mysqldump -u root -p你的密码 inventory > /data/backup/inventory_$DATE.sql
find /data/backup -mtime +30 -name "*.sql" -delete
EOF
```

给执行权限：

```bash
chmod +x /data/backup.sh
```

设置每天凌晨 3 点自动备份：

```bash
crontab -e
```

添加一行：

```
0 3 * * * bash /data/backup.sh
```

保存退出（nano 编辑器按 Ctrl+X → Y → Enter）。

### 恢复数据库

```bash
docker exec -i inventory-mysql mysql -u root -p你的密码 inventory < /data/backup/inventory_备份日期.sql
```

## 常见错误

### Q: 连不上服务器
**A:** 检查 IP 是否正确，服务器是否开机。阿里云/腾讯云默认屏蔽 22 端口的话，去控制台安全组里放行 22 端口。

### Q: `git clone` 报错
**A:** 网络问题。多试几次，或者配置代理。

### Q: `docker compose up -d` 报错 / 卡住
**A:** 检查磁盘空间：`df -h`。如果 /data 分区满了，清理一下。另外国内网络访问 Docker Hub 和 Maven 仓库可能慢，第一次需要耐心等。

### Q: 浏览器打不开管理后台
**A:** 
1. 先用 `docker compose ps` 确认所有容器都是 `Up` 状态
2. 去云服务商安全组放行 **80 端口**（HTTP）和 **8080 端口**（API）
3. 如果放行了还不行，运行 `docker compose logs server` 看后端有没有报错

### Q: 图片上传报 413
**A:** 文件超过 10MB 了，压缩一下再上传。

### Q: 数据库要定期清理吗
**A:** 系统设计为长期运行，不需要日常清理。上传的图片会占用磁盘，定期检查 `/data/inventory/data/uploads` 目录大小。

### Q: 如果要迁移服务器
**A:** 
1. 运行 `bash /data/backup.sh` 手动备份数据库
2. 把 `/data/backup/` 和 `/data/inventory/data/uploads/` 拷贝到新服务器
3. 在新服务器部署代码后，恢复数据库
